机器视觉硬件如何选型

Posted on 2025-08-07
  • LZQ's Blog
Home

机器视觉硬件选型是为特定的机器视觉应用需求，从众多硬件组件中科学地挑选出最合适的相机、镜头、光源、处理器及其他辅助设备的过程。其核心目标是构建一套能稳定、精确、高效地完成指定视觉任务（如检测、测量、识别、定位等）的硬件系统。

# 核心组件选型

  1. 相机

    • 类型：
      • 面阵相机： 最常用，适用于静止或连续运动的物体成像（如传送带）。选型重点：分辨率、帧率、传感器尺寸、像元尺寸、快门类型。
      • 线阵相机： 适用于高速连续运动物体或需要超高分辨率的场景（如卷材检测、印刷检测）。选型重点：行频、分辨率（线数）、传感器长度、像元尺寸、行曝光时间控制。
      • 区域扫描相机： 即面阵相机。
      • 3D 相机： 用于获取深度信息（如体积测量、引导抓取）。技术包括：双目立体视觉、结构光、激光三角测量、飞行时间。选型重点：精度、量程、点云密度、抗环境光能力、速度。
    • 传感器技术：
      • CMOS： 主流技术，性价比高，速度快，功耗低，抗拖影能力好，支持全局快门或卷帘快门。
      • CCD： 成像质量（尤其是低照度、均一性）通常优于 CMOS，但速度慢、功耗高、成本高、易产生拖影，逐渐被 CMOS 取代。仍有部分高端应用使用。
    • 分辨率：
      • 根据检测精度要求计算。所需最小像素数 = (待测特征尺寸 / 检测精度要求) * 2 （满足奈奎斯特采样定理）。例如，要检测 0.1mm 的缺陷，精度要求 0.05mm，则单个特征至少需要 (0.1 / 0.05) * 2 = 4 个像素覆盖。再结合视野大小计算所需总像素。
      • 考虑视野大小。
      • 避免过度追求高分辨率：增加成本、降低帧率、增大数据量、需要更强的镜头和处理能力。
    • 帧率：
      • 根据产线速度和检测时间窗口计算。所需最小帧率 = (物体移动速度 / 相机视野内允许的最大物体位移) 。同时考虑图像处理算法所需时间。
      • 高帧率需求：高速运动物体检测、需要捕捉快速瞬变过程。
    • 快门类型：
      • 全局快门： 所有像素同时曝光，适合拍摄运动物体，无运动模糊（在曝光时间内）。工业应用首选。
      • 卷帘快门： 像素逐行曝光。拍摄运动物体时会产生果冻效应。成本低，常用于消费级或对运动模糊不敏感的场景。
    • 接口：
      • GigE Vision： 主流，性价比高，线缆长（可达 100m），带宽适中（1Gbps -> 10Gbps -> 25Gbps -> 100Gbps）。
      • USB3 Vision： 即插即用方便，带宽高（5Gbps），但线缆短（<5m），供电能力有限，稳定性要求高时需谨慎。
      • Camera Link： 传统高速接口，带宽高（可达 850MB/s），但需要专用采集卡，线缆短（<10m），成本高。逐渐被 CoaXPress 取代。
      • CoaXPress： 新兴高速接口，带宽极高（单根同轴可达 12.5Gbps，多通道聚合可达 50Gbps），线缆长（可达 100m+），抗干扰强，支持 PoCXP 供电。成本较高。
      • MIPI： 主要用于嵌入式系统（如手机、平板），工业级相机较少直接使用。
    • 色彩：
      • 黑白： 灵敏度高，分辨率高（相同传感器尺寸下），速度快，成本低。适用于尺寸测量、有无检测、OCR 等对颜色不敏感的应用。
      • 彩色： 用于需要区分颜色的应用（如颜色检测、表面缺陷颜色识别）。需注意拜耳滤镜带来的分辨率和灵敏度损失。真彩色相机通常指彩色相机。
    • 传感器尺寸： 影响视野和景深（与镜头组合）。常见尺寸有 1/4", 1/3", 1/2.5", 1/2", 2/3", 1", 1.1", 1.2" 等。大尺寸传感器通常灵敏度更高，像元更大，但相机和镜头成本也更高。
    • 像元尺寸： 影响分辨率和灵敏度。小像元分辨率潜力高，但单个像元进光量少，低照度性能可能较差；大像元灵敏度高，但相同传感器尺寸下分辨率低。
    • 灵敏度 / 量子效率： 在低光照条件下尤其重要。查看相机的 QE 曲线（量子效率 vs 波长）和信噪比指标。
    • 动态范围： 反映相机同时捕捉亮部和暗部细节的能力。高动态范围对于明暗对比强烈的场景很重要（如逆光、反光表面）。
    • 其他特性： 触发输入 / 输出、可编程控制、多 ROI、LUT、Binning/Skipping、温度稳定性、防护等级。

  2. 镜头

    • 焦距： 决定视野和工作距离。 焦距 f ≈ (传感器尺寸 * 工作距离) / 视野 。需要根据视野和工作距离计算所需焦距范围。
    • 光圈：
      • 大光圈： 进光量大，适合低照度环境，但景深小。
      • 小光圈： 景深大，适合有一定厚度或不在同一平面的物体，但进光量少。通常选择可调光圈的镜头。
      • F 值越小，光圈越大。
    • 接口： 必须与相机接口匹配。常见工业接口：C 口（最普遍），CS 口（需转接环用于 C 口相机），F 口（大靶面），M42，M58 等。
    • 分辨率： 镜头的解析能力必须匹配相机的分辨率（传感器像元尺寸）。镜头分辨率通常用 MTF 曲线表示。选择镜头分辨率应略高于传感器需求（Nyquist 频率处 MTF 值 > 10%-20%）。
    • 畸变： 镜头使图像产生的几何变形（桶形、枕形）。高精度测量应用需选择低畸变镜头（<0.1%）。
    • 类型：
      • 定焦镜头： 最常见，性价比高。
      • 变焦镜头： 焦距可调，灵活性高，但通常分辨率、畸变控制不如同等级定焦镜头，成本高。适用于需要频繁调整视野的应用。
      • 远心镜头： 精密测量必备！** 消除透视误差，提供恒定的放大倍率，景深内物体尺寸测量不受距离影响。成本高，尺寸大。
      • 微距镜头： 专为高倍率、短工作距离设计。
      • 液态镜头： 通过电压控制焦距，实现毫秒级自动对焦，适用于工作距离变化或物体高度不平的应用。
    • 匹配传感器尺寸： 镜头成像圈必须大于或等于相机传感器尺寸，否则图像边缘会有暗角或模糊。

  3. 光源

    • 目的： 突出被测特征，抑制背景干扰，提高对比度，保证成像稳定性和一致性。光源是视觉系统成败的关键！

    • 类型：

      • LED 光源： 绝对主流。寿命长、功耗低、响应快、亮度 / 颜色可选、发热量小。常见形态：

      • 环形光： 通用性好，提供漫射光，减少阴影。

      • 条形光： 用于长条状物体或需要定向照明。可组合使用。

      • 背光源： 轮廓检测、透明物体杂质检测。

      • 同轴光： 检测光滑平面（如镜面、金属）上的划痕、凹凸。

      • 穹顶光： 提供均匀漫射光，消除反光，用于复杂曲面、反光物体。

      • 点光源 / 线光源： 用于特殊角度照明或配合线阵相机。

      • 结构光： 投射特定图案（如条纹、网格）用于 3D 测量。

      • 卤素灯 / 荧光灯： 亮度高，光谱连续，但发热量大、寿命短、亮度稳定性差，已基本被 LED 取代。

    • 颜色： 选择能最大化目标特征与背景对比度的颜色。常用颜色：红、蓝、绿、白、红外、紫外。

      • 互补色原理： 物体颜色与光源颜色互为补色时，物体吸收最多光，看起来最暗。
      • 单色光： 常能提高对比度（尤其对黑白相机）。红外 / 紫外用于特殊检测（如穿透材料、荧光物质）。

    • 亮度 / 强度： 需足够亮以满足曝光时间和图像信噪比要求。考虑光源衰减，留有余量。

    • 照明方式：

      • 明场照明： 光线反射进入镜头，观察物体表面反射特性。
      • 暗场照明： 光线以低角度照射，光滑表面反射光不进入镜头（呈暗色），粗糙表面或缺陷散射光进入镜头（呈亮色），用于检测划痕、凹凸、纹理。
      • 透射照明： 即背光。

    • 控制器： 用于调节亮度（模拟 / PWM）、开关控制、频闪控制（与相机曝光同步）。高频闪是高速应用的关键。

  4. 图像采集卡

    • 作用： 对于需要 Camera Link 或 CoaXPress 接口的相机，采集卡是必需的。它负责将相机的高速图像数据传输到 PC 内存。
    • 选型：
      • 接口类型和带宽必须匹配相机（Camera Link Base/Medium/Full/Deca, CoaXPress 1.0/2.0/6.25/12.5）。
      • 通道数：支持单相机或多相机。
      • 内存：板载内存大小影响传输稳定性和丢帧率。
      • I/O 支持：触发输入 / 输出、编码器输入等。
      • 软件兼容性：与视觉软件库兼容。
    • 趋势： 随着 GigE Vision 和 USB3 Vision 的普及，以及 PC 接口（如 PCIe）带宽的提升，独立采集卡的使用在减少。

  5. 处理器 / 工控机

    • CPU： 核心数量、主频对图像处理速度至关重要。选择性能强劲的多核处理器（Intel Core i5/i7/i9, Xeon）。
    • 内存： 足够大以容纳操作系统、视觉软件、图像数据和处理中间结果。推荐 16GB 起步，复杂应用或高分辨率 / 高帧率应用需要 32GB 甚至更多。
    • 存储： SSD 固态硬盘（NVMe 更好）用于快速启动、加载程序和存储图像 / 结果。
    • GPU： 对于深度学习应用、复杂的 3D 点云处理、实时渲染、某些加速算法 **** 非常关键。选择具备足够 CUDA 核心和显存的 NVIDIA 专业显卡（如 Quadro, RTX 系列）。
    • 接口： 提供足够数量和类型的接口连接相机（GigE, USB3）、光源控制器、I/O 设备、显示器、网络等。
    • 操作系统： Windows 10/11 IoT Enterprise LTSC (长期稳定，工业推荐)， Linux (开源，定制性强)。
    • 可靠性： 工业环境需考虑宽温、抗振动、防尘、无风扇设计（或良好散热）。
    • 外形： 工控机（Box PC）、嵌入式无风扇工控机、机架式服务器等。

  6. 其他硬件

    • 光学滤镜：
      • 偏振片： 消除眩光、反光，增强表面特征对比度。
      • 带通 / 窄带滤光片： 只允许特定波长（如与光源匹配）的光通过，抑制环境光干扰。
      • 中性密度滤光片： 均匀减弱所有波长的光强，防止过曝。
    • 传感器 / 编码器： 用于提供物体位置信号触发相机 / 光源，实现精确位置同步。
    • I/O 模块： 连接 PLC、传感器（光电开关、接近开关）、执行器（气缸、继电器）等，实现与自动化设备的交互。
    • 防护罩 / 机械支架： 保护相机、镜头、光源，确保安装稳固、位置可调。
    • 线缆与连接器： 选择高质量、屏蔽良好的线缆和可靠的工业连接器，保证信号传输稳定，减少干扰。

# 关键选型考量因素

  1. 应用需求
    • 检测目标是什么？ (尺寸、缺陷、有无、定位、识别、计数、读码、3D 测量等)
    • 精度要求？ (亚像素级？微米级？毫米级？)
    • 速度要求？ (产线节拍？每分钟检测多少件？)
    • 视野大小？
    • 工作距离？ (相机到被测物体的距离)
    • 被测物体特性？ (尺寸、形状、材质、颜色、表面反光性、纹理、运动状态 - 静止 / 连续运动 / 间歇运动 / 高速运动)
    • 环境条件？ (温度、湿度、粉尘、油污、水汽、振动、光照干扰 - 阳光 / 其他设备光)
  2. 性能指标
    • 基于应用需求推导出的具体参数：分辨率、帧率、景深、曝光时间、信噪比、动态范围等。
  3. 成本预算
    • 在满足性能要求的前提下，平衡各组件成本。避免过度设计，也切忌因节省成本导致系统无法满足要求。
  4. 系统集成与兼容性
    • 各组件（相机接口、镜头接口、光源控制器接口、软件 SDK）之间的兼容性。
    • 与现有自动化设备（PLC、机器人）的通信接口（Ethernet/IP, Profinet, Modbus TCP, RS232, I/O）。
    • 视觉软件的选择（Halcon, VisionPro, OpenCV, LabVIEW Vision, 国产如海康 MVS, 迈德威视等）也会影响硬件选型（如对特定相机品牌 / 接口的支持，是否需要特定采集卡）。
  5. 可靠性与维护
    • 工业环境的严苛性要求硬件具有高可靠性和长寿命（尤其 LED 光源寿命）。
    • 考虑维护便利性（如镜头更换、清洁）。
  6. 供应商支持与服务
    • 选择有良好技术支持和售后服务的供应商至关重要，尤其是在调试和后期维护阶段。

# 选型流程建议

  1. 明确定义需求： 详细列出所有应用要求、目标和约束条件。
  2. 计算核心参数： 根据视野、精度计算所需分辨率；根据速度和视野计算所需帧率；估算工作距离和景深。
  3. 初步选择相机： 基于分辨率、帧率、接口、传感器类型 / 尺寸、快门类型等筛选。
  4. 选择镜头： 基于相机传感器尺寸、工作距离、视野计算焦距；根据精度要求决定是否需要远心镜头；根据景深和光照条件选择光圈范围；确保镜头分辨率匹配相机。
  5. 设计光源方案： 根据被测物特征和环境，确定光源类型、颜色、照明方式（明场 / 暗场 / 背光）、安装角度。通常需要实验验证多种方案。
  6. 选择处理器 / 工控机： 根据算法复杂度、图像数据量（分辨率 x 帧率 x 通道数）、是否用深度学习选择 CPU/GPU 和内存配置。
  7. 选择其他组件： 如相机接口需要，选采集卡；选光源控制器；选滤镜、编码器、I/O 模块等。
  8. 验证与测试： 在条件允许的情况下，尽可能搭建原型系统进行实际测试，验证选型是否能满足所有要求（精度、速度、稳定性）。进行环境适应性测试。
  9. 评估供应商与成本： 综合评估各供应商方案的技术指标、价格、供货期、技术支持和服务能力。
  10. 最终决策与采购。

# 一个简化选型案例（手机屏缺陷检测）

  • 需求： 检测 6 英寸手机屏表面划痕、脏污、亮点 / 暗点，精度要求 0.05mm 缺陷，产线速度 1 秒 / 片。
  • 计算：
    • 视野：略大于屏幕尺寸，约 180mm x 90mm。
    • 精度：0.05mm 缺陷 -> 至少需要 4 个像素覆盖 -> 单方向最小像素数 = (180mm / 0.05mm) * 2 = 7200 像素 (宽度方向)。选择 800 万像素相机 (例如 3296 x 2472) 可能足够或有冗余。
    • 帧率：一片屏检测时间 < 1 秒。假设相机一次拍摄覆盖整个屏（可能需要多工位或多相机），则曝光 + 传输 + 处理时间 < 1 秒。选择 30fps 相机比较宽裕。
  • 相机选型： 高分辨率面阵 CMOS 全局快门相机，800 万像素以上（如 12MP）， GigE 或 USB3 接口，黑白（缺陷检测通常对颜色不敏感）。
  • 镜头选型： 远心镜头（保证边缘和中心测量一致）或低畸变定焦镜头。根据传感器尺寸（如 1.1"）和视野计算焦距（如～50mm）。选择高分辨率镜头（MTF 满足要求）。
  • 光源选型： 针对光滑玻璃表面，常用同轴光或低角度条形光（暗场） 来突出划痕、脏污。亮点 / 暗点检测可能需要均匀面光源（明场）。可能需要组合照明或分步检测。常选白色或红色 LED。
  • 处理器： 性能较强的工控机（i7 CPU, 32GB RAM），如果需要深度学习辅助检测缺陷，则需配备 GPU (如 NVIDIA RTX 3060 以上)。
  • 其他： 编码器触发相机抓拍， I/O 与 PLC 通信报告结果。
